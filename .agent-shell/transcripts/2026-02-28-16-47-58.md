现在清晰了！让我格式化一下 AST 结构来分析：

```
expression_statement
├── print("hello")    → (identifier) (selector (argument_part (arguments ...)))
├── foo               → (identifier) 单独
├── foo.bar           → (identifier) (selector (unconditional_assignable_selector (identifier)))
├── foo.bar()         → (identifier) (selector (...)) (selector (argument_part (arguments)))
└── MyClass.staticMethod() → 同上
```

**关键发现**：

| 代码 | AST 结构 | 区分特征 |
|------|----------|----------|
| `print("hello")` | `identifier` + `selector > argument_part` | 有 `argument_part` = 函数调用 |
| `foo` | `identifier` 单独 | **无法确定** - 可能是变量引用、getter、或无参函数调用 |
| `foo.bar` | `identifier` + `selector > unconditional_assignable_selector` | 属性访问 |
| `foo.bar()` | `identifier` + `selector` + `selector > argument_part` | 方法调用 |

**核心问题**：

当前代码（第 362-363 行）把 `expression_statement` 下的裸 `identifier` 一律标记为 `font-lock-function-call-face`：

```elisp
(expression_statement
 (identifier) @font-lock-function-call-face)
```

这是**错误的**，因为：
1. `foo;` 这样的裸 identifier 可能只是变量引用（虽然在 Dart 中无意义，但语法上合法）
2. 真正的函数调用应该有 `argument_part`

**正确的函数调用模式应该是**：

```elisp
;; 函数调用：identifier 后跟 argument_part
(expression_statement
 (identifier) @font-lock-function-call-face
 (selector (argument_part)))

;; 或者用 :anchor 确保 argument_part 直接跟在 identifier 后面
```

## 最终解决方案

**策略**：使用两个规则，让变量规则优先匹配（更具体），通过 `:override` 覆盖函数调用规则。

### 修改 dart-ts-mode.el

将当前的规则替换为：

```elisp
;; 方法调用中的变量 - identifier 后跟属性 selector（优先匹配）
:language 'dart
:feature 'variable
:override t
'((expression_statement
   (identifier) @font-lock-variable-use-face
   (selector [(unconditional_assignable_selector)
              (conditional_assignable_selector)])))

;; 直接函数调用 - identifier 后跟 argument_part
:language 'dart
:feature 'function
'((expression_statement
   (identifier) @font-lock-function-call-face
   (selector (argument_part))))
```

**原理**：
1. `ref.onDispose()` - identifier `ref` 后跟 `unconditional_assignable_selector` → 匹配变量规则 ✓
2. `print("hello")` - identifier `print` 后跟 `argument_part` → 不匹配变量规则，匹配函数规则 ✓

两个规则都会匹配 `ref.onDispose()` 中的 `ref`，但变量规则有 `:override t`，所以它优先。

你需要确保 `variable` feature 在 `function` feature 之后定义（或者给 variable 规则加 `:override t`），这样覆盖才能生效。
